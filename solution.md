# Shabdonkaghar - Project Solution & Architecture

This document provides a comprehensive overview of the `shabdonkaghar` project, detailing its technology stack, project structure, and the purpose of key files. It is designed to help new developers understand the architecture and start contributing quickly.

## 🚀 Tech Stack

The project is built using a modern, full-stack React ecosystem powered by Vite and TanStack.

### Core Technologies
*   **React 19**: The latest version of the popular UI library.
*   **TypeScript**: For strong typing and better developer experience.
*   **Vite**: A blazing fast frontend tooling and development server.

### Routing & Data Fetching
*   **TanStack Router** (`@tanstack/react-router`): A fully type-safe routing library for React.
*   **TanStack Start** (`@tanstack/react-start`): A full-stack React framework leveraging TanStack Router.
*   **TanStack Query** (`@tanstack/react-query`): For powerful asynchronous state management and data fetching.

### Styling & UI
*   **Tailwind CSS (v4)**: Utility-first CSS framework for rapid UI development.
*   **Radix UI** (`@radix-ui/react-*`): Unstyled, accessible UI components.
*   **shadcn/ui**: Reusable UI components built on top of Radix UI and Tailwind CSS (inferred from `clsx`, `tailwind-merge`, and `class-variance-authority`).
*   **Lucide React**: Beautiful and consistent iconography.

### Forms & Validation
*   **React Hook Form**: Performant, flexible, and extensible forms with easy-to-use validation.
*   **Zod**: TypeScript-first schema declaration and validation.

## 📄 Key Files and Their Purpose

### `src/routes/`
This is where your pages and layouts live. TanStack Router uses a file-based routing system. Any file added here will automatically generate a route in `routeTree.gen.ts`.
*   `__root.tsx`: The root layout of the application that wraps all other routes.
*   `index.tsx`: The home page route (`/`).

### `src/router.tsx`
Initializes the TanStack Router instance using the generated `routeTree`. It sets up default preloads and route context.

### `src/server.ts` & `src/start.ts`
These files form the backbone of the **TanStack Start** integration.
*   `server.ts` handles the server-side rendering (SSR), API requests, and server startup.
*   `start.ts` is responsible for hydrating the React application on the client side.

### `src/components/ui/` (Inside components)
Typically contains the primitive UI components generated by shadcn/ui (e.g., Button, Input, Dialog). These are meant to be customized directly in the source code.

### `package.json` Scripts
*   `npm run dev`: Starts the Vite development server with HMR.
*   `npm run build`: Builds the application for production.
*   `npm run lint`: Runs ESLint to check for code issues.
*   `npm run format`: Formats code using Prettier.

## 🛠️ Getting Started for Developers

1.  **Install dependencies**: The project uses Bun as its package manager (indicated by `bun.lock`).
    ```bash
    bun install
    ```
2.  **Start the development server**:
    ```bash
    bun run dev
    ```
3.  **Adding Routes**: Create a new file in `src/routes/` (e.g., `src/routes/about.tsx`). TanStack Router will automatically update `src/routeTree.gen.ts`.
4.  **Adding Components**: If using shadcn/ui, add new primitives using the CLI, or create custom functional components in `src/components/`.

This structure promotes a clear separation of concerns, strong type safety across the network boundary, and an excellent developer experience.
